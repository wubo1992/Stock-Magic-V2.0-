## 这个系统是什么

一个运行在你电脑上的程序。它每天（或按需）分析你设定的股票池，告诉你：
**哪只股票、在什么条件下、应该考虑买入或卖出。**

它不帮你下单。它只给你一张"信号清单"，最终决定权在你手里。

---

## 系统的输入和输出

### 输入
- 你指定的股票列表（在 `config.yaml` 里设置）
- 这些股票的历史价格数据（从互联网自动获取，免费）
- 你设置的策略参数（在 `config.yaml` 里）

### 输出
每次运行后，生成一张信号表，格式如下：

| 时间 | 股票 | 信号 | 强度 | 触发原因 | 参考止损 |
|------|------|------|------|----------|----------|
| 2026-03-03 | NVDA | 买入 | ★★★★ | 突破60日高点，成交量放大2倍 | $105.20 |
| 2026-03-03 | AAPL | 观望 | — | 无有效信号 | — |

同时保存到 `output/signals.csv` 文件，方便追踪历史信号。

## 系统的两种运行模式

### 模式 1：回测模式（Backtest）
用过去几年的历史数据"假装"运行策略，看看如果当时用这个策略，结果如何。
这是验证策略是否靠谱的方式。

**运行命令：**
```bash
python main.py --mode backtest --start 2019-01-01 --end 2024-12-31
```

### 模式 2：实盘信号模式（Live Signal）
对最新数据运行策略，输出当前是否有信号。
每天开盘前或收盘后运行一次。

**运行命令：**
```bash
python main.py --mode live
```

## 什么是"事件驱动"，用大白话说

想象一个工厂流水线：

1. **原材料到来**（新的K线数据）→ 触发流水线启动
2. **第一道工序**（数据处理器）→ 把数据清洗成标准格式
3. **第二道工序**（策略引擎）→ 计算各种指标，判断条件是否满足
4. **第三道工序**（信号生成器）→ 如果条件满足，产生一个信号
5. **成品出库**（输出模块）→ 把信号打印出来或存到文件

每一道工序只做自己的事，通过传递"工单"（事件）来协调。
这种设计的好处：每道工序可以独立测试，出了问题容易定位。

## 数据来源

使用 `yfinance`（Yahoo Finance 的 Python 接口），完全免费，无需注册账号。
可以获取：日线、周线、分钟线历史数据，以及基本面数据。

局限性：
- 分钟级数据只有最近 60 天
- 数据质量不如付费数据源，但对日线策略完全够用
- 不支持盘前盘后数据

**未来可升级为：** Alpaca（免费账户支持实时数据）或 Polygon.io（付费）

## 策略的选择原则

Claude Code 在 Phase 0 提出策略候选时，必须满足：

1. **有实证支持**：策略背后的逻辑有学术论文或经过大量实盘验证
2. **可解释**：能用一两句话说清楚为什么这个信号有意义
3. **适合日线级别**：不需要盯盘，每天运行一次就够
4. **有明确的进出场规则**：不模糊，条件可以量化
5. **有内置的风险控制**：每个信号必须附带止损参考价

## 不在本系统范围内的事情

- ❌ 自动下单执行
- ❌ 期权、期货、杠杆产品的信号
- ❌ 高频交易（分钟级以下）
- ❌ 加密货币
- ❌ 仓位管理（系统不知道你有多少钱）
- ❌ 税务或合规建议
