## 1. 系统是什么

这是一个**美股交易信号系统**，基于 Mark Minervini《股票魔法师》中的 SEPA + VCP 策略。

**系统做什么：**
- 每天扫描股票池（88只手动配置 + 自动抓取新闻热门股）
- 对每只股票检查 SEPA + VCP 策略的所有条件
- 输出买入/卖出信号，格式为：股票代码、信号方向、强度、触发原因、参考止损

**系统不做什么：**
- 不自动下单
- 不连接券商账户
- 不管理真实资金

> **信号系统的定位：给你参考依据，最终决策由你来做。**

## 2. 策略原理：SEPA + VCP

### 2.1 整体逻辑

策略分两层筛选：

```
第一层：趋势模板（大背景对不对？）
    ↓ 通过
第二层：VCP 形态（当前时机好不好？）
    ↓ 通过
入场：枢轴点突破 + 大成交量确认
```

类比：先找好城市（趋势对），再找好地段（VCP蓄力），最后等到好时机下手（突破）。

---

### 2.2 第一层：趋势模板（8个条件）

这8个条件确保只买「正在强势上涨」的股票，不买下跌中的股票。

| 条件 | 含义 |
|------|------|
| 收盘价 > SMA50 | 价格在短期均线上方 |
| 收盘价 > SMA150 | 价格在中期均线上方 |
| 收盘价 > SMA200 | 价格在长期均线上方 |
| SMA50 > SMA150 > SMA200 | 三线多头排列（短 > 中 > 长） |
| SMA150 > SMA200 | 中期均线高于长期均线 |
| SMA200 近20天处于上升 | 长期趋势本身在向上走 |
| 收盘价 ≥ 52周低点 × 1.25 | 距最低点至少涨了 25% |
| 收盘价 ≥ 52周高点 × 0.75 | 距最高点不超过 25%（仍接近高位） |

> **通俗理解**：这8个条件合在一起，就是 Minervini 说的「Stage 2 上升阶段」——股票已经从底部起来，正处于趋势最强的阶段，不是低位横盘，也不是高位崩盘。

---

### 2.3 第二层：VCP 形态（波动收缩形态）

VCP（Volatility Contraction Pattern）是 Minervini 策略**最核心**的入场过滤器。

**VCP 是什么：**

股票在上涨途中会做「整理」——短暂回调后继续涨。VCP 的特征是：每次回调的幅度越来越小，成交量也越来越少。

```
价格走势示意图：
         ●
        / \
       /   \       ●
      /     \     / \
     /       \   /   \         ●
    /         \ /     \       / \
   /           ●       \     /   \          ← 突破！买入点
  /                     \   /     ●
 /                       \ /
●                         ●

第1次回调：-15%，成交量大    第2次：-9%，量减少    第3次：-4%，量极少
```

**为什么有效：**
- 每次回调量减少 → 说明卖盘在枯竭，想卖的人越来越少
- 最终一根大阳线突破 → 卖盘彻底消失，买盘压倒性涌入

**系统如何检测 VCP（算法）：**
1. 取突破日之前 50 天作为底部窗口
2. 用「3-bar 对称法」找局部高点和低点（一个高点两侧各有3根更低的K线，才算局部高点）
3. 配对高低点，算出每次回调的深度
4. 验证：回调深度递减 + 每次回调均量递减 + 最近10天箱体宽度 < 8%

---

### 2.4 第三层：入场触发（条件10-11）

| 条件 | 具体规则 |
|------|----------|
| 枢轴点突破 | 收盘价超过过去30天最高收盘价，且超过幅度 > 0.5% |
| 成交量放大 | 当天成交量 ≥ 20日平均成交量的 1.5 倍 |

> 两个条件同时满足才发信号。成交量放大是关键——没有量的突破往往是假突破。

---

### 2.5 相对强度过滤（RS，条件9）

计算该股票过去12个月的涨幅，与股票池所有股票的涨幅比较，只保留**排名前30%**的股票（RS ≥ 70%）。

> 类比：只选班级里成绩前30名的学生，不管排名靠后的。

---

### 2.6 卖出条件（三种，任意一种触发即卖出）

**条件一：固定止损（-10%）**
```
收盘价 ≤ 买入价 × 90% → 立即卖出
```
作用：防止单笔大亏。

**条件二：追踪止盈（从最高点回落 -20%）**
```
收盘价 ≤ 持仓期间最高价 × 80% → 卖出
```
作用：保住利润。股价涨得越高，止盈线也跟着抬高。
举例：买入 $100 → 涨到 $150（最高点）→ 跌回 $120（$150×80%）→ 卖出，锁定 +20% 盈利。

**条件三：时间止损**
```
持仓 ≥ 20天，且盈利 < 3% → 卖出
```
作用：释放资金，不让资金被「不涨不跌」的股票占用。

---

### 2.7 信号强度评分（1-5星）

| 星级 | 条件 |
|------|------|
| ★★★★★ | 成交量 ≥ 2倍均量 且 突破幅度 > 1.5% |
| ★★★★ | 成交量 1.5-2倍 或 突破幅度 1-1.5% |
| ★★★ | 刚好达到最低标准 |
| +1星 | RS 排名 ≥ 90%（额外加分） |

## 3. 系统架构

```
signal_system/
├── main.py              ← 运行入口（实盘/回测模式）
├── config.yaml          ← 所有参数配置（不改代码只改这个）
│
├── data/
│   ├── fetcher.py       ← 数据获取（Alpaca IEX → Yahoo Finance备用 → 缓存）
│   └── cache/           ← 本地缓存（.pkl格式，避免重复下载）
│
├── strategy/
│   └── sepa_minervini.py ← 核心策略（11个进场条件 + 3个出场条件 + VCP）
│
├── backtest/
│   └── engine.py        ← 回测引擎（时间机器，按历史日期逐日模拟）
│
├── signals/
│   └── generator.py     ← 信号输出（格式化 + 保存CSV）
│
├── events/
│   ├── base.py          ← 事件基类
│   └── queue.py         ← 事件总线
│
└── output/
    └── signals.csv      ← 历史信号记录（自动追加）
```


### config.yaml 关键参数

```yaml
strategy:
  # 均线周期
  sma_short: 50
  sma_mid: 150
  sma_long: 200

  # RS 过滤
  rs_min_percentile: 70    # 只选涨幅前30%的股票

  # VCP 参数
  vcp_lookback: 50         # 检测VCP的回看天数
  vcp_min_contractions: 2  # 最少几次收缩（建议2-4）
  vcp_final_range_pct: 0.08 # 末端箱体最大宽度（8%）

  # 突破参数
  pivot_lookback: 30       # 枢轴点回看天数
  min_breakout_pct: 0.005  # 最小突破幅度（0.5%）
  volume_mult: 1.5         # 成交量倍数要求

  # 出场参数
  stop_loss_pct: 0.10      # 固定止损（-10%）
  trailing_stop_pct: 0.20  # 追踪止盈（从最高点-20%）
  time_stop_days: 20       # 时间止损天数
  time_stop_min_gain: 0.03 # 时间止损最低盈利要求（3%）
```

## 6. 信号格式说明

### 6.1 输出格式

```
日期         股票    信号   强度   触发原因                                    参考止损
2026-03-04   NVDA   买入   ★★★★★  [趋势] SMA50(800)>SMA150(650)>SMA200(600)，
                                  SMA200上升中，距52W低点+85%，距52W高点-3% |
                                  [RS] 相对强度排名92% |
                                  [VCP] VCP3T：回调18%→11%→5%，量能收缩，末端3.2%箱体 |
                                  [突破] 超过30日高点$201.02，幅度+3.0% |
                                  [量能] 成交量1.8倍均量              $186.33
```

### 6.2 日期含义

**信号日期 = 策略检测到条件满足的当天**。

实际操作建议：**第二天开盘买入**（因为当天K线要等收盘才确认）。

### 6.3 触发原因的四个模块

| 模块 | 对应条件 | 包含信息 |
|------|----------|----------|
| `[趋势]` | 条件1-8 | 三条均线数值、SMA200趋势、距52周高低点幅度 |
| `[RS]` | 条件9 | 相对强度排名百分位 |
| `[VCP]` | VCP形态 | 收缩次数、各次回调幅度、末端箱体宽度 |
| `[突破]` | 条件10 | 枢轴点价格、突破幅度 |
| `[量能]` | 条件11 | 当日成交量是均量的几倍 |

## 7. 重要概念解释

### 7.1 样本内 vs 样本外

用考试打比方：

- **样本内（练习题）**：用来调参数、验证逻辑的历史数据。因为"对着答案做题"，表现天然好看。
- **样本外（真正的考试）**：从未接触过的数据，完全靠策略自身逻辑应对。这才是对未来实盘表现的真实预估。

本项目把总区间按 **2:1** 分割：
```
2020-01-01 ──────────────── 2024-02-11 ──────── 2026-03-03
         样本内（前2/3）            样本外（后1/3）
```

### 7.2 夏普比率

衡量「每承担一单位风险，获得多少回报」。

- < 1.0：回报相对风险偏低
- 1.0~2.0：良好
- > 2.0：优秀

### 7.3 盈亏比

平均盈利笔数 / 平均亏损笔数（按金额比较）。

举例：盈亏比 3.25 = 每亏 $1，平均能赚 $3.25。
即使胜率只有 28%，只要盈亏比够高，长期依然盈利：
```
10笔交易中：3笔盈利 × $3.25 + 7笔亏损 × (-$1) = $9.75 - $7 = +$2.75（净盈利）
```

### 7.4 最大回撤

持仓净值从最高点到最低点的最大跌幅。衡量「最坏情况下你会亏多少」。

- < 20%：优秀
- 20-30%：可接受
- > 30%：心理压力较大

### 7.5 追踪止盈 vs 固定止损

| | 固定止损 | 追踪止盈 |
|--|----------|----------|
| 参考点 | 买入价（固定不动） | 持仓最高价（随股价上涨而抬高） |
| 触发条件 | 跌破买入价 -10% | 从最高点下跌 -20% |
| 作用 | 防止大亏 | 保住涨幅 |

## 8. 常见问题解答

**Q：信号出来后会自动下单吗？**

不会。系统只输出信号到终端和 CSV 文件，所有买卖操作由你手动执行。

**Q：止损是 -10%，为什么有些交易亏了 -15%？**

这是「日线回测的跳空缺口风险」。策略每天只在收盘时检查一次价格。如果股票当天直接跳空低开并收于 -15%，系统才在收盘时发现并卖出，此时只能以 -15% 成交，而不是 -10%。

实盘中可以挂限价止损单来规避，但大幅跳空依然无法完全避免。

**Q：为什么有的交易明明涨了很多，最后却以小盈利或微亏出场？**

这是**追踪止盈**的工作机制。以 NVDA 为例：

```
买入 $103.81 → 涨到最高 $135.57（+30.6%）
→ 追踪止盈线跟到 $135.57 × 80% = $108.46
→ 实际以 $103.56 出场（-0.2%）
```

说明股价从最高点 $135 回落了超过 20%，触发了追踪止盈。虽然最终没赚到钱，但这个机制是正确的——在不知道股价会跌多深的情况下，用追踪止盈来保护已有利润是合理的选择。

**Q：每月信号数只有 1 个，太少了怎么办？**

两个方向：
1. **扩大股票池**：在 `config.yaml` 的 `manual_universe` 里加入更多股票（当前 88 只）
2. **适当放宽 VCP 参数**：
   ```yaml
   vcp_min_contractions: 1   # 从2次降到1次
   vcp_final_range_pct: 0.12  # 从8%放宽到12%
   ```

**Q：KLAC 被买了三次都亏了，能不能不买它？**

可以，在 `config.yaml` 里从 `manual_universe` 删掉 `KLAC` 即可。但建议先回测确认删掉后整体结果改善，再做决定。

**Q：回测结果很好，实盘会一样吗？**

不会完全一样，原因：
1. 当前使用 Alpaca IEX 数据，成交量只覆盖约 20% 的真实市场成交量，成交量信号可能偏差
2. 回测假设每次信号都以当天收盘价成交，实盘会有买卖价差和冲击成本
3. 回测不考虑仓位管理，实盘每笔只会用部分资金
