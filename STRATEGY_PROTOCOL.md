# STRATEGY_PROTOCOL.md — 策略研究协议

> 这个文件规定了 Claude Code 如何研究、提出、验证和迭代交易策略。
> 在写任何策略代码之前，必须完成这里规定的研究流程。

---

## 为什么策略研究要有协议

交易策略的失败大多不是因为代码写错，而是因为：
- 策略本身没有经过严格验证就上线
- 过度优化历史数据（过拟合），到真实市场就失效
- 没有考虑策略在不同市场环境下的表现差异

这个协议是为了让我们系统性地避免这些错误。

## Phase 0 完整流程

### Step 0.1 — 策略候选提案

Claude Code 必须提出 **2-3 个策略候选**，每个候选包含以下内容：

```
策略名称：（简洁的名字）
核心逻辑：（用一句话说清楚，不用术语）
详细原理：（2-3段，解释为什么这个策略应该有效）
有效性依据：（引用学术研究或知名交易员的实践经验）
适用市场环境：（在什么情况下效果好，什么情况下失效）
主要风险：（最大的2-3个弱点是什么）
适合我们的理由：（为什么适合从零开始、日线级别、美股）
```

### Step 0.2 — 用户选择

用户看完提案后，选择一个策略（或要求修改方案）。
Claude Code 不能在用户确认之前开始写代码。

### Step 0.3 — 策略细化

用户确认后，Claude Code 将所选策略细化为**精确的规则**：

```
进场条件（缺一不可）：
  条件1：...
  条件2：...
  条件3：...

出场条件（满足任一即出场）：
  止损：当价格跌破 X 时
  止盈：当价格涨到 Y 时 / 当 Z 条件出现时
  时间止损：持仓超过 N 天且无盈利时

信号强度评分标准：
  ★★★★★（最强）：所有条件强度均超标准的 50% 以上
  ★★★★：所有条件满足
  ★★★：核心条件满足，次要条件勉强
  ★★：仅供关注，暂不行动

参数列表（将写入 config.yaml）：
  参数名: 默认值  # 含义说明
```

用户确认规则后，Claude Code 才开始 Phase 1 编码。

## 回测标准（Phase 2）

### 必须测试的时间段

| 时间段 | 市场特征 | 目的 |
|--------|----------|------|
| 2019年全年 | 牛市 | 验证策略能否抓住上涨 |
| 2020年3月 | 极端暴跌 + 快速反弹 | 验证策略在黑天鹅中的表现 |
| 2022年全年 | 熊市 + 高通胀 | 验证策略在下跌市的损失控制 |
| 2023-2024年 | 样本外验证（OOS）| 最关键，验证策略不是过拟合 |

**原则：2019-2022 用于"训练"策略参数，2023-2024 用于"考试"。**
考试用的数据不能用来调整参数。

### 必须输出的回测指标

Claude Code 必须计算并用中文解释以下每一个指标：

| 指标 | 及格线 | 含义 |
|------|--------|------|
| 胜率 | > 40% | 发出的信号中，最终盈利的比例 |
| 盈亏比 | > 1.5 | 平均每笔盈利 / 平均每笔亏损 |
| 最大回撤 | < 25% | 从最高点到最低点的最大跌幅 |
| 信号频率 | 每月 2-10 个 | 太少说明策略太保守，太多质量存疑 |
| 年化收益率 | > 15% | 按年计算的总体收益 |
| 夏普比率 | > 1.0 | 收益 / 风险的综合评分 |

**如果样本外（2023-2024）任意一项不达标，必须向用户汇报，并提出原因分析和改进方向，而不是直接修改参数再测。**

### 回测报告格式

```
═══════════════════════════════════
回测报告 — [策略名称]
测试区间：2019-01-01 至 2024-12-31
股票池：[列出测试的股票]
═══════════════════════════════════

【样本内 2019-2022】
总信号数：XX 个
胜率：XX%（超过40%及格）
盈亏比：X.X（超过1.5及格）
最大回撤：XX%（低于25%及格）
年化收益：XX%

【样本外 2023-2024 ← 这才是真正的考试】
总信号数：XX 个
胜率：XX%  ← 与样本内差距 X%
盈亏比：X.X
最大回撤：XX%
年化收益：XX%

【结论】
策略整体评价：合格 / 需要优化 / 建议换策略
最大弱点：...
建议下一步：...
═══════════════════════════════════
```

## 策略迭代规则（Phase 3）

### 允许优化的事情
- 调整指标参数（比如把20日均线改为50日）
- 增加一个过滤条件（比如加入成交量确认）
- 调整止损止盈规则

### 不允许的优化
- 针对某段特定历史数据"专门"调参数
- 因为某次大亏损而专门加入一个针对性规则
- 在看了样本外结果之后，再去修改参数

以上行为会导致"过拟合"——策略在历史数据上表现很好，但到了真实市场就失效。

每次优化后，Claude Code 必须：
1. 解释这次优化改变了什么
2. 解释为什么这个改变有逻辑支持
3. 重新运行完整回测
4. 对比优化前后的所有指标

## 策略候选参考库

以下是 Claude Code 在 Phase 0 可以考虑的策略方向（仅供参考，不限于此）：

**趋势跟随类**
- 均线突破（价格突破N日均线）
- 动量策略（过去N个月涨幅最大的股票继续买入）
- 52周新高突破

**均值回归类**
- RSI 超卖反弹
- 布林带边界反弹

**事件驱动类**
- 财报后价格跳空（earnings gap）
- 成交量异动信号
- 新高确认信号（价格创新高 + 成交量放大）

**Claude Code 在提案时必须解释：为什么这个策略有效的理论基础是什么，而不只是说"历史数据表现好"。**
